# Обновление списков (без пересборки Podkop)

После того как форк Podkop уже знает ваши community-теги, обновление контента делается в репозитории списков.

## Где лежат данные

- Домены секции:
  - `lists/<tag>/domains.srs`
- Подсети секции:
  - `lists/<tag>/subnets.srs`

## Процесс обновления

1. Обновляете сервисы в `services/` (при необходимости).
2. Пересобираете агрегированные секции:
   - `scripts/build-group-lists.ps1`
3. Проверяете статистику:
   - `scripts/analyze-database.ps1`
4. Коммитите изменения в `main`.

Podkop подтянет новые данные по своему расписанию `update_interval` или по `podkop list_update`.

## Важно

- Имя секции (`<tag>`) должно совпадать с тем, что добавлено в `COMMUNITY_SERVICES` форка Podkop.
- Формат файлов:
  - `domains.srs` — по одному домену в строке
  - `subnets.srs` — по одной подсети CIDR в строке
